登录组件

引用

<script src='http://qa.developer.163.com/component/login/login.js'></script>

登录方法

  • NTESCommonLogin.submitLogin 登录方法
  • NTESCommonLogin.Login 提供UI的登录方法

辅助功能方法

  • NTESCommonLogin.refresh 获取用户当前状态
  • NTESCommonLogin.Suggestion 添加自动完成，邮箱后缀自动填充

NTESCommonLogin.submitLogin

//登录方法
NTESCommonLogin.submitLogin(data,callback);

参数

  • data {username,password,type,product,savelogin}
  • 参数名 类型 描述
    username String 提供的用户名.
    password String 提供的用户密码
    type Number 0表示password经过md5加密，对用户密码做MD5前需要对密码中的'和\做转换，其前再加上一个\字符。1表示password为明文【1】
    product String 产品标识【163.com】
    savelogin Number 是否保留用户登录信息，设置cookie:NTES_PASSPORT。1保留，0不保留【0】
  • callback {Function} - 登录成功后调用的函数传入userInfo callback(userInfo)

NTESCommonLogin.Login

//提供UI的登录方法
NTESCommonLogin.Login(callback,options);

//默认无UI时，此方法会创建出一段代码结构.
//但是需要使用者自定义样式.
//参考引用样式 http://qa.developer.163.com/component/login/login.css

参数

  • callback {Function} - 登录成功后调用的函数传入userInfo callback(userInfo)
  • options {obj,top,left,template,mask,center} 非必要参数
  • 参数名 类型 描述
    obj Object 参考对象的坐标值作为登录框的显示位置.
    top Number 当有obj存在的前提，对于top值的偏移量
    left Number 当有obj存在的前提，对于left值的偏移量
    template string 可以将登录容器中的结构传入，但是一定要保留form中的所有表单.不能遗漏
    mask Boolean 是否存在蒙板层【true】
    center Boolean 是否居中显示【true】
    boxId String 登录框容器的ID【component_js_N_login_wrap】
    openSuggestion Boolean 是否启用自动完成. true|flase【true】
    display String 默认登录容器的display属性【block】

NTESCommonLogin.Suggestion

//自动完成填充用户类型
Passport = new NTESCommonLogin.Suggestion();
Passport.bind(Element);
//每一个被实例化出来的对象只能绑定一个对象实体
//demo中可以有多个自动感应对象

参数

  • Element {Element} - 需要自动完成的input Dom 对象 比如：Passport.bind(document.getElementById("xxx"));

demo

NTESCommonLogin.refresh

//获取用户当前状态
var UserInfo = NTESCommonLogin.refresh();

返回值

  • UserInfo {userName,userMail,userDomain,userIp,userLogin}
  • 参数名 类型 描述
    userName String 用户名.
    userMail String 用户邮箱名
    userDomain String 邮箱主域 如：163.com
    userIp string 例如：61.135.255.84.1373258834864378 用户IP + 登录时间戳【 】
    userLogin Boolean 用户是否登录状态【false】

补充：

功能性登录组件

完全脱离依赖，不需要任何类库支持

简单介绍：

1、通过iframe进行通讯.登录结果由iframe的url返回,所以需要设置主域..

2、域名必须为163.com下的产品才能使用此组件

3、必须传入callback,否则不进行后续操作

4、如果需要测试，必须自己模拟一个163.com的域名
